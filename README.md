## 🎯 概述

这个导航网站可以同时管理 **Markdown 文档** 和 **HTML 网页**，左侧列表显示所有文件，右侧展示内容。

### 界面布局
- **左上**：分类切换按钮（📄 站内文档 / 🌐 站内网站）
- **左下**：文件列表（显示可用文件的标题）
- **右侧**：内容展示区（渲染后的文档或网页）

---

## 📂 文件目录结构

```
根目录/
├── index.html          ← 导航主页（你正在用的）
├── md/                 ← 放 Markdown 文档
│   ├── 01.md
│   ├── 02.md
│   └── ...
└── html/               ← 放 HTML 网页
    ├── 01.html
    ├── 02.html
    └── ...
```

---

## 📝 如何添加 Markdown 文档

### 第一步：创建文件
在 `/md/` 目录下新建一个文件，命名为 `01.md` ~ `100.md`（两位数字编号）。

### 第二步：写标题
文件**第一行**必须是标题，格式为 `# 你的标题`，这个标题会显示在左侧列表中。

**示例：**
```markdown
# 这是我的第一个文档

正文内容从这里开始...
```

如果第一行不是 `#` 开头，系统会自动使用文件名作为标题。

### 第三步：写内容
标题之后可以自由使用 Markdown 语法编写内容。

---

## 🌐 如何添加 HTML 网页

### 第一步：创建文件
在 `/html/` 目录下新建一个文件，命名为 `01.html` ~ `100.html`。

### 第二步：设置标题
修改 `<title>` 标签的内容，这个会显示在左侧列表中。

**示例：**
```html
<!DOCTYPE html>
<html lang="zh">
<head>
    <meta charset="UTF-8">
    <title>我的网页标题</title>
</head>
<body>
    <h1>网页内容</h1>
</body>
</html>
```

### 第三步：编写页面
编写完整的 HTML 页面，包括样式和脚本，会完整展示在右侧。

## 🛠️ 故障排查

### 文件没有出现在列表中？
1. 检查文件是否在正确的目录（`/md/` 或 `/html/`）
2. 检查文件名格式是否正确：
   - ✅ `01.md`、`05.md`、`99.md`
   - ❌ `1.md`、`abc.md`、`01.txt`
3. 检查文件编号是否在 `01~100` 范围内
4. 刷新浏览器页面

### 标题显示不对？
- **MD 文件**：检查第一行是否以 `# ` 开头
- **HTML 文件**：检查 `<title>` 标签内容

### 链接打不开或显示为文本？
- 确保链接格式正确：`[文字](https://完整网址)`
- 网址必须包含 `https://` 或 `http://`

### 图片不显示？
- 检查图片地址是否正确
- 确保图片文件存在且可访问

## 📋 检查清单

添加新文件时，确认以下几点：

- [ ] 文件放在正确的目录（`/md/` 或 `/html/`）
- [ ] 文件名是两位数字 + 正确扩展名（如 `03.md`）
- [ ] MD 文件第一行是 `# 标题`
- [ ] HTML 文件有正确的 `<title>` 标签
- [ ] 所有链接都包含完整网址（`https://`）
- [ ] 刷新了浏览器页面

## 💡 最佳实践

1. **编号规划**：用不同编号段管理不同类型内容
2. **标题清晰**：使用简洁明确的标题方便查找
3. **链接完整**：所有链接使用绝对路径或完整 URL
4. **测试验证**：添加文件后刷新页面确认显示正常
